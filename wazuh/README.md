# Wazuh Security Monitoring Stack

A single-node [Wazuh](https://wazuh.com/) deployment using Docker Compose, including:

- **Wazuh Manager** — SIEM/XDR engine and agent server
- **Wazuh Indexer** — OpenSearch-based data store
- **Wazuh Dashboard** — Web UI built on OpenSearch Dashboards

## Directory layout

```
wazuh/
├── config/
│   ├── certs/                       # TLS certificates (generated by setup.sh)
│   ├── wazuh-indexer/
│   │   ├── opensearch.yml           # Indexer configuration
│   │   └── internal_users.yml       # Indexer users (hashed passwords)
│   ├── wazuh-manager/
│   │   └── ossec.conf               # Manager main configuration
│   └── wazuh-dashboard/
│       └── opensearch_dashboards.yml
├── docker-compose.yml
├── .env.example
├── setup.sh
└── README.md
```

## Quick start

### Prerequisites

| Tool | Notes |
|------|-------|
| Docker ≥ 24 | With the Compose plugin (`docker compose`) |
| `openssl` | For certificate generation |
| Linux host | Required for the `vm.max_map_count` kernel setting |
| ≥ 4 GB RAM | OpenSearch is memory-intensive |

### 1. Run the setup script (recommended)

```bash
cd wazuh
bash setup.sh
```

The script will:
1. Copy `.env.example` → `.env` (first run only).
2. Prompt you to review and update passwords.
3. Set `vm.max_map_count=262144` (required by OpenSearch).
4. Generate self-signed TLS certificates for all components.
5. Start the stack with `docker compose up -d`.

### 2. Manual setup (alternative)

```bash
cp .env.example .env
# Edit .env — change all default passwords

# Set kernel parameter
sudo sysctl -w vm.max_map_count=262144

# Generate certificates (see setup.sh generate_certs function for reference)

# Start the stack
docker compose up -d
```

## Environment variables

| Variable | Description | Default |
|----------|-------------|---------|
| `WAZUH_VERSION` | Image tag for all Wazuh images | `4.11.2` |
| `INDEXER_PASSWORD` | OpenSearch `admin` user password | `SecretPassword_ChangeMe!` |
| `WAZUH_API_USERNAME` | Wazuh API username | `wazuh-wui` |
| `WAZUH_API_PASSWORD` | Wazuh API password | `MyS3cr37P450r.*-` |
| `DASHBOARD_USERNAME` | Dashboard backend user | `kibanaserver` |
| `DASHBOARD_PASSWORD` | Dashboard backend password | `kibanaserver_ChangeMe!` |
| `DASHBOARD_PORT` | Host port for the dashboard | `443` |
| `INDEXER_HEAP` | JVM heap for the Indexer (50% of RAM) | `512m` |

> **Security:** Change all default passwords in `.env` before deploying to production.

## Accessing the dashboard

After the stack has started (allow ~2-3 minutes for first-time initialisation):

```
https://<your-server-ip>:<DASHBOARD_PORT>
```

Default credentials:
- **Username:** `admin`
- **Password:** value of `INDEXER_PASSWORD` in your `.env`

## Connecting agents

### Linux/macOS agent

```bash
# Install the Wazuh agent
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo gpg --dearmor -o /usr/share/keyrings/wazuh.gpg
echo "deb [signed-by=/usr/share/keyrings/wazuh.gpg] https://packages.wazuh.com/4.x/apt/ stable main" | sudo tee /etc/apt/sources.list.d/wazuh.list
sudo apt update && sudo apt install -y wazuh-agent

# Configure and enroll
sudo WAZUH_MANAGER="<your-server-ip>" WAZUH_AGENT_NAME="$(hostname)" \
     dpkg-reconfigure wazuh-agent

sudo systemctl enable --now wazuh-agent
```

### Windows agent

Download the MSI installer from https://packages.wazuh.com/4.x/windows/ and set the manager IP during installation.

## Useful commands

```bash
# View all logs
docker compose logs -f

# View a specific service
docker compose logs -f wazuh-manager

# Stop the stack
docker compose down

# Stop and remove all data (WARNING: destructive)
docker compose down -v

# Check indexer cluster health
curl -sku admin:"${INDEXER_PASSWORD}" https://localhost:9200/_cluster/health | python3 -m json.tool
```

## Upgrading

To upgrade to a newer Wazuh release:

1. Update `WAZUH_VERSION` in `.env`.
2. Pull the new images and recreate the containers:

```bash
docker compose pull
docker compose up -d
```
